# Audience Analysis 활용 사례 {#analyze-audiences-use-cases}

Audience Analysis를 사용하면 Customer Journey Analytics에서 Experience Platform 대상 멤버십 데이터를 보고할 수 있습니다. 이는 다른 매개 변수 및 구성 세부 사항과 함께 수집 중인 프로필 데이터 세트를 결정하는 데 도움이 되는 Audience Analysis 구성 마법사를 통해 관리되는 구성을 통해 달성할 수 있습니다. 자세한 개요 정보는 [대상 분석 개요](/help/connections/audience-analysis/audience-analysis-overview.md)를 참조하십시오.

이 문서에는 Audience Analysis에서 제공하는 값을 강조 표시하는 예제 사용 사례가 포함되어 있습니다. 사용 사례를 검토하기 전에 먼저 아래 보고 고려 사항을 숙지하십시오. 사용 사례는 보고서의 최종 출력에 영향을 미칠 수 있으므로 이러한 고려 사항을 염두에 두어야 합니다.

## 보고 고려 사항

Audience Analysis의 초기 릴리스는 Experience Platform 대상을 Customer Journey Analytics으로 처리하고 수집하는 데 필요한 필수 기반 작업을 수립합니다. 분석을 수행할 때 Workspace 프로젝트 전반에 걸쳐 결과에 영향을 줄 수 있는 몇 가지 요인에 유의해야 합니다.

* **대상 멤버십 데이터는 이전 날(&quot;어제&quot;)에 대해서만 정확합니다**: 대상 멤버십 데이터에는 통합 프로필 서비스에서 생성한 최신 프로필 스냅숏 데이터 세트가 항상 포함됩니다. 이 프로필 데이터 세트는 일별 스냅샷이며 전날(&quot;어제&quot;)에 대해서만 정확하며, 매일 밤 자동으로 재생성되고 다시 처리됩니다. 대상 차원은 기록 대상 상태를 재구성하는 데 사용할 수 없고, 보고 및 분류에 사용할 수 있습니다.

   * 예: 선택한 보고 기간에 관계없이 CJA 보고 가능한 대상은 항상 수집된 최신 프로필 스냅샷(&quot;어제&quot;)에 있는 대상 멤버십 상태를 준수합니다.

      * 보고 기간 을 &quot;최근 30일&quot;로 확장하는 경우 더 많은 이벤트가 포함되며 대상 크기가 변경되고 있다는 인상을 줍니다. 그러나 대상자의 프로필 구성은 선택한 시간대에 관계없이 항상 &quot;어제&quot;의 스냅샷과 일치합니다.

* **차원에 포함할 해당 이벤트가 있어야 합니다**: Audience Analysis 차원은 해당 이벤트가 CJA에 있는 곳에서만 분석할 수 있습니다. 동작, 채널 또는 라이프사이클 순간이 CJA 연결에서 이벤트로 표시되지 않으면 분석할 수 없습니다.

   * 예: 광고를 통해 사람을 타깃팅하는 데 사용되는 대상에는 CJA 대상보다 RTCDP 대상에 있는 훨씬 더 많은 사람이 포함됩니다. CJA 대상자는 보고 기간 동안 CJA에서 이벤트를 가진 사람으로 제한되기 때문입니다.

* **ID 확인은 단일 네임스페이스만을 기반으로 합니다**: ID 확인은 Audience Analysis 구성의 일부로 선택한 ID 네임스페이스에 전적으로 의존합니다. 분석은 해당 ID 네임스페이스로 제한되며 그 밖에 있는 이벤트는 대상 분석 보고에 사용할 수 없습니다.

   * 예: CRM과 ECID를 결합하는 결합된 이벤트 데이터 세트와 CRM ID를 사용하는 대상 분석 구성의 경우, CRM ID가 포함된 행만 CJA에서 보고 가능한 대상의 일부로 인식됩니다. 따라서 결과 대상자 크기는 예상보다 작을 수 있습니다.

## 예시 사용 사례

### 사용 사례 1

특정 대상자가 주어진 채널(예: 웹 또는 앱)에서 어떻게 동작하는지 파악하여 다음과 같은 질문에 답변합니다.

* _&quot;가치가 높은 잠재 고객의 구성원이 현재 사이트에서 수행하는 작업(페이지, 기능, 단계)은 무엇입니까?&quot;_

* _&quot;이 대상의 캠페인 및 콘텐츠 색인 초과(다른 모든 대상과 비교)&quot;_

#### 구성 흐름

1. 단일 ID 네임스페이스(예: ECID 또는 CRM_ID) 및 웹 중심의 데이터 보기에 대해 CJA에서 Audience Analysis를 구성합니다.

   * 이렇게 하면 일일 프로필 스냅숏 내보내기를 통해 대상 멤버십 데이터를 선택한 연결로 자동으로 수집합니다

   * 이벤트 데이터 세트에서 적용 범위가 가장 높다고 생각되는 ID 네임스페이스를 선택하는 것이 좋습니다

1. Workspace 프로젝트를 빌드하여 다음과 같은 작업을 수행합니다.

   * 페이지, 제품, 캠페인, 장치 등을 기준으로 대상 이름 분류

   * 세션, 전환율, 사람당 매출과 같은 지표에 대해 대상과 비대상(또는 다른 대상과) 비교

1. 생성된 인사이트를 사용하여 채널 최적화 전략(예: 타깃팅 규칙, 콘텐츠 또는 오퍼 조정)을 미세 조정합니다.

### 사용 사례 2

마케터 또는 여정 디자이너가 어떤 대상이 겹치는지 파악하여 중복을 제거하고 캠페인 간 오퍼 충돌을 방지할 수 있도록 지원합니다.

* _&quot;충성도 멤버, 장바구니 포기 사용자 및 높은 이탈 성향 간에 현재 얼마나 겹치나요?&quot;_

* _&quot;이번 주에 고부가가치 프로모션 오퍼에 충돌할 가능성이 가장 높은 대상은 무엇입니까?&quot;_

#### 구성 흐름

1. CJA/AJO 활성화에 맞춰 정렬된 단일 ID 네임스페이스(예: 여정이 사람 기반인 경우 CRM_ID)에 대해 RTCDP에서 Audience Analysis를 구성합니다.

   * 이렇게 하면 일일 프로필 스냅샷 내보내기를 통해 선택한 연결로 대상자 멤버십 데이터가 자동으로 수집됩니다.

   * 이를 사용하여 이미 키 참여 및 전환 보고를 지원하는 기존 데이터 보기를 보강할 수 있습니다.

1. 기본 제공 대상 분석 개요 템플릿 및 오버랩 시각화 사용:

   * 대상 교차 및 오버인덱싱/언더 인덱싱 보기 (예: 로열티 골드에 있는 장바구니 포기 사용자의 %)를 실행합니다.

   * 슬라이스는 핵심 차원(예: 장치 유형, 제품 관심사)별로 겹쳐서 충돌이 가장 중요한 위치를 파악합니다.

1. 이 통찰력을 사용하여 다음과 같은 중요한 측면을 미세 조정합니다.

   * RTCDP 및 AJO의 오퍼 충돌 규칙 또는 마케팅 작업 규칙입니다.

   * 대상 세분화(예: 겹치는 부분이 너무 큰 대상 정의 조이기).

### 사용 사례 3

최근 주요 대상을 떠난 고객의 비헤이비어와 해당 종료 주변에서 수행한 작업을 이해하여 다음과 같은 질문에 답변합니다.

* _&quot;주요 대상자를 떠난 사용자는 누구이며, 해당 대상자가 종료 시 수행한 작업은 무엇입니까?&quot;_

* _&quot;종료하기 직전에 발생한 작업 (오류, 낮은 참여, 가격 변경).&quot;_

#### 구성 흐름

1. 단일 ID 네임스페이스(예: CRM_ID 또는 로그인 ID) 및 관련 데이터 보기(웹, 앱, CRM 등)에 대해 CJA에서 Audience Analysis를 구성합니다.

   * 이렇게 하면 일일 프로필 스냅샷 내보내기를 통해 선택한 연결에 대상 멤버십 데이터가 자동으로 수집됩니다.

   * 이벤트 데이터 세트에서 적용 범위가 가장 크다고 생각되는 ID 네임스페이스를 선택하는 것이 좋습니다.

1. 대상 분석/대상 개요 템플릿(_어제_(으)로 수정됨)에서 다음을 사용합니다.

   * 현재 구성원: 아직 대상자에 속한 사용자

   * 종료한 대상자: 어제 퇴사한 사람

1. Workspace 프로젝트를 빌드하여 다음과 같은 작업을 수행합니다.

   * 어제 Audience X를 종료한 프로필로 필터링한 다음 다음을 확인합니다.

      * 종료까지의 비헤이비어(마지막 세션, 오류, 가격/오퍼 노출, 채널 혼합).

      * 종료 후 작동(제품 전환, 다운그레이드, 휴면 상태 전환).

   * 종료된 집단을 지역, 장치, 기간, 가치 계층별로 분류하여 영향력이 큰 주머니를 찾습니다.

1. RTCDP 또는 AJO의 여정 업데이트 및 Win-back 대상 구성에 Insights를 사용하십시오.
