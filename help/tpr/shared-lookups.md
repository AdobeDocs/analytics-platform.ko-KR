# 공유 조회

Customer Journey Analytics에서 조회 데이터 세트는 추가 컨텍스트를 통해 이벤트 데이터를 강화합니다. 예를 들어 구매 이벤트에 제품 이름, 카테고리 및 가격을 추가하는 제품 카탈로그 데이터 세트입니다. 또는 마케팅 이벤트에 캠페인 세부 사항을 추가하는 캠페인 메타데이터 데이터 세트입니다.

조회를 사용하면 이벤트 자체에 저장되지 않은 특성을 사용하여 이벤트 데이터에 대해 보고할 수 있습니다.
일반적으로 조회 데이터 세트는 고정된 단일 경로를 통해 이벤트에 연결됩니다. 이벤트 데이터 세트의 키 필드가 조회 데이터 세트의 하나의 키 필드와 일치합니다. 이 조회는 두 데이터 세트를 연결하는 한 가지 방법만 있을 때 작동하지만 이 간단한 링크는 일반적인 실제 시나리오에서 분류됩니다.

* 이벤트 소스에 따라 제품 SKU 또는 제품 ID의 이벤트에 연결된 제품 카탈로그입니다.
* 사용자 특성 조회는 채널에 따라 다른 ID 네임스페이스의 이벤트에 연결됩니다(웹 이벤트의 경우 이메일, 매장 이벤트의 경우 로열티 ID).
* 직접(개인별) 및 간접적으로(B2B 보고 목적으로 계정별) 이벤트에 연결된 프로필 데이터 세트

공유 조회를 사용하면 조회 데이터 세트와 조회 데이터가 증가하는 이벤트 간에 여러 조인 경로를 정의할 수 있으므로 제한된 고정 경로 조인을 해결할 수 있습니다. 각 경로는 조회 행을 이벤트 행에 일치시키는 한 가지 방법을 설명합니다. 조회를 기반으로 구축된 차원 또는 지표는 사용할 경로를 선택할 수 있습니다. 이제 동일한 조회 데이터 세트가 단일 구성의 여러 보고 시나리오에 전원을 공급할 수 있습니다.

공유 조회는 [총 모집단 보고](./tpr.md)의 기초이기도 하며, 공유 조회를 사용하여 프로필 데이터 세트를 이벤트에 연결합니다.

## 개념

다음 섹션에서는 주요 공유 조회 개념을 설명합니다.

### 경로 결합

조인 경로는 조회 데이터 세트와 이벤트 간의 행을 일치시키는 단일 경로입니다. 각 조인 경로는 다음과 같습니다.

* **경로 이름**&#x200B;입니다. 사용자가 읽을 수 있는 레이블로, 차원 및 지표를 작성할 때 UI에서 경로를 식별하는 데 사용됩니다.
* 이벤트 쪽의 **키 필드**. 이 필드는 이벤트를 조회 데이터와 일치시키는 데 사용됩니다.
* 조회 쪽의 **일치하는 키 필드**&#x200B;입니다.  이 필드는 키가 일치하는 필드입니다.
* 선택적 **네임스페이스**&#x200B;입니다. 키 필드가 ID 맵인 경우 네임스페이스가 필요합니다.

단일 조회 데이터 세트에는 하나 이상의 조인 경로가 있을 수 있습니다. 해당 조회의 필드에 빌드된 차원 및 지표는 사용할 경로를 지정할 수 있습니다. 경로를 지정하지 않으면 데이터 세트의 기본 경로가 사용됩니다.

### 컨테이너별 일치

프로필 데이터 세트(총 모집단 보고와 함께 사용)의 경우 공유 조회는 컨테이너 유형에 따라 조인을 자동으로 구성하는 컨테이너별 일치 설정을 지원합니다.

* **개인 컨테이너별 일치**. 조회는 이벤트 데이터 세트의 ID 맵을 키로 사용하여 개인 ID를 통해 이벤트에 결합됩니다.
* **계정 컨테이너별로 일치** [!BADGE B2B edition]{type=Informative url="https://experienceleague.adobe.com/ko/docs/analytics-platform/using/cja-overview/cja-b2b/cja-b2b-edition" newtab=true tooltip="Customer Journey Analytics B2B Edition"}. 조회는 계정 ID를 통해 연결됩니다.
* **글로벌 계정 컨테이너별로 일치**([!BADGE B2B edition]{type=Informative url="https://experienceleague.adobe.com/ko/docs/analytics-platform/using/cja-overview/cja-b2b/cja-b2b-edition" newtab=true tooltip="Customer Journey Analytics B2B Edition"} 및 글로벌 계정이 활성화됨). 조회는 글로벌 계정 ID를 통해 연결됩니다.

컨테이너별 일치는 주요 필드를 수동으로 구성할 필요 없이 일반적인 사례를 처리합니다. 컨테이너별 일치의 주요 장점은 중복 제거를 자동으로 처리한다는 것입니다. 컨테이너에는 고유한 ID(개인, 계정 또는 글로벌 계정의 경우)가 저장됩니다.

총 모집단 보고 외에 컨테이너별 일치 를 사용하여 다른 조회 데이터 세트에 대한 조인 경로를 정의할 수도 있습니다.

### 필드별 일치

또는 필드별로 프로필 데이터 세트를 일치시킬 수 있습니다. 이 일치는 특정 ID를 기반으로 이벤트 데이터의 각 이벤트에 대한 직접 조회를 생성합니다. 필드별 일치를 사용하는 경우 결과에 중복 데이터가 포함될 수 있으므로 특히 지표와 함께 사용할 경우 혼동을 주는 결과가 발생할 수 있습니다. 자세한 내용은 [예제](#example)를 참조하십시오.

### ID가 키 필드로 매핑됨

조인 양쪽의 키 필드가 ID 맵(네임스페이스가 여러 개인 ID가 포함된 필드)인 경우 추가 구성이 필요합니다.

* **기본 키** 또는 **네임스페이스**. ID 맵의 기본 키를 사용하거나 특정 네임스페이스를 선택하여 일치시킬 수 있습니다. 네임스페이스를 선택하는 것이 더 일반적인 선택입니다. 기본 키가 모든 프로필 데이터 소스에 채워지지 않습니다.
* **보조 네임스페이스**. 기본 네임스페이스가 결합된 데이터 세트에서 공통되는 주어진 행에 채워지지 않는 경우 대체 네임스페이스를 지정할 수 있습니다. 조인은 채울 때 기본 네임스페이스를 사용하고 그렇지 않으면 보조 네임스페이스로 폴백합니다.
* **경로 간 일관성**. 연결에 대해 동일한 ID 맵이 여러 공유 조회에서 키 필드로 사용되는 경우 네임스페이스 선택 사항은 해당 조회 간에 일관되어야 합니다.

### 조회 경로 조회

조회 데이터 세트 자체는 다른 조회 데이터 세트에 조인할 수 있습니다. 조회에 대한 이 조회는 2수준 조회 체인(이벤트 → 조회 A → 조회 B)을 생성합니다.

조회 체인의 각 수준에는 고유한 조인 경로가 있을 수 있습니다. 두 번째 수준 조회의 필드를 기반으로 구축된 차원 또는 지표는 각 단계에서 구성된 경로를 사용하여 체인을 트래버스합니다. 두 수준보다 더 세부적인 조회 체인은 지원되지 않습니다.


## 사용 시기

다음 중 하나에 해당할 경우 공유 조회를 사용합니다.

* 두 가지 이상의 방법으로 동일한 조회 데이터 세트를 이벤트에 가입해야 합니다.
* 서로 다른 이벤트가 서로 다른 ID 네임스페이스를 사용하는 B2C(비즈니스 대 소비자) ID 데이터로 작업합니다.
* 사용자 및 계정 모두에 이벤트를 연결해야 하는 B2B(business to business) 연결을 구성합니다.
* 프로필 데이터 세트를 연결에 추가하여 총 모집단 보고를 수행할 수 있습니다.

조회 데이터 세트에 하나의 분명한 조인 키가 있고 조회 데이터 세트의 데이터를 이벤트에 연결하는 한 가지 방법만 필요한 경우 단일 경로를 구성할 수 있습니다. 공유 조회는 이러한 간단한 사례도 지원합니다.
