# 총 모집단 보고

총 모집단 보고에서는 프로필 및 조회 데이터 세트에 정의된 엔터티와 이벤트 데이터 세트의 시간 기반 이벤트 시리즈 이상에 대해 분석하고 보고할 수 있는 기능을 도입했습니다. 이 기능을 사용하면 비즈니스 고객 기반의 전체 범위를 반영하는 새로운 클래스의 쿼리, 지표 및 대상 정의를 사용할 수 있습니다.

Customer Journey Analytics은 이벤트를 중심으로 구축됩니다. 모든 지표, 모든 시각화, 모든 패널, 모든 보고서는 날짜 시간 범위와 해당 날짜 시간 범위 동안 발생하는 이벤트에 고정됩니다. 다음과 같은 솔루션에 대해 질문할 수 있습니다.

| 질문 | 이벤트 | 날짜 시간 범위 |
|---|---|---|
| 지난주에 몇 명이 구매했나요? | 구매 | 지난 주 |
| Q1에 몇 개의 고객이 가격 페이지를 방문했습니까? | 방문 | Q1 |

작업 영역 패널의 날짜 시간 범위는 데이터를 필터링하고 차원 및 지표는 해당 날짜 시간 범위 동안 발생한 상황을 설명합니다.

그러나 귀사에서 반드시 답해야 할 질문은 발생한 문제에 관한 것이 아닙니다. 때로는 인구 자체에 대해 알아야 할 필요가 있습니다.

| 질문 | 이벤트 | 날짜 시간 범위 |
|---|---|---|
| 현재 활성화된 고객은 몇 명입니까? | 해당 사항 없음 | 해당 사항 없음 |
| 데이터베이스에 계정이 몇 개 있습니까? | 해당 사항 없음 | 해당 사항 없음 |
| 지난 30일 동안 구매하지 않은 회원은 몇 명입니까? | 해당 없음 | 지난 30일 |

이러한 질문은 이벤트가 아닌 실제 사용자와 계정에 대한 질문입니다. 이러한 사용자 또는 계정이 최근에 작업을 수행했는지 여부에 상관없이 존재합니다.

총 모집단 보고에서는 프로필 데이터에 대해 보고하는 새로운 지표 클래스를 도입합니다. 프로필 데이터 세트의 개인 및 계정을 포함할 수 있는 해당 프로필 데이터는 패널의 날짜 범위와 독립적입니다. 전체 모집단 보고를 사용하면 동일한 분석에서 모집단 기반 지표와 이벤트 기반 지표를 혼합하여 고객이 누구이며 어떤 작업을 수행했는지 보다 포괄적으로 파악할 수 있습니다.

총 모집단 보고를 사용하면 Customer Journey Analytics에서 이벤트 활동에 관계없이 프로필 및 조회 데이터 세트에 정의된 모든 엔터티에 대해 보고할 수 있습니다. 이 보고에는 다음이 포함됩니다.

* **프로필 기반 쿼리**: 프로필(모든 사람, 계정, 기회 및 구매 그룹)의 특성(이벤트에 관계없이)을 분석합니다.
* **프로필 - 이벤트 쿼리**: 보고 기간 동안 특정 작업 또는 경험을 수행하지 않은 프로필(모든 사람, 계정, 기회 및 구매 그룹)을 식별합니다.
* **공유 조회**: 수집 비용을 줄이고 성능을 개선하기 위해 여러 엔터티 간에 조회 데이터 세트를 다시 사용할 수 있도록 지원합니다.

## 총 모집단 보고 지표

총 인구 보고 지표는 Customer Journey Analytics에서 일반적으로 사용되는 지표와 다르게 동작합니다.

* 총 모집단 보고 지표는 패널 날짜 범위에 바인딩되어 있지 않습니다. **[!UICONTROL 총 사용자(프로필)]**&#x200B;과(와) 같은 총 모집단 보고 지표는 패널에 적용된 날짜 범위와 관계없이 프로필 데이터 세트의 현재 모집단을 반환합니다. 날짜 필터 및 날짜 범위 비교는 이러한 필터 및 비교가 이벤트 지표에 영향을 주는 방식처럼 총 모집단 보고 지표에 영향을 주지 않습니다.
* 총 모집단 보고에는 연결에 프로필 데이터 세트가 필요합니다. 총 모집단 보고 지표는 연결에 하나 이상의 이벤트 데이터 세트와 함께 하나 이상의 프로필 데이터 세트가 포함된 경우에만 표시됩니다. 이벤트 데이터 세트만 있는 연결은 이전과 동일하게 계속 작동하며 총 인구 보고 지표를 표시하지 않습니다.

Workspace에서 총 인구 지표는 패널의 날짜 범위를 준수하는 지표와 준수하지 않는 지표를 빠르게 식별할 수 있도록 고유 아이콘(TBD)으로 표시됩니다. 총 모집단 지표는 대부분의 위치에서 이벤트 지표와 함께 사용할 수 있지만 이벤트 시퀀스에 의존하는 시각화 유형(예: 폴아웃 및 플로우)은 지원되지 않습니다.

### 표준 총 인구 보고 지표

기본적으로 시스템에는 세 개의 표준 총 인구 보고 지표가 포함되며, 이 지표는 프로필 데이터 세트가 연결되어 있는 모든 데이터 보기에서 사용할 수 있습니다.

* **총 사용자(프로필)**: 프로필 데이터 세트에 있는 총 사용자 수입니다.
* **총 계정(프로필)**: 프로필 데이터 집합에 있는 총 계정 수입니다. [!BADGE B2B Edition]{type=Informative url="https://experienceleague.adobe.com/ko/docs/analytics-platform/using/cja-overview/cja-b2b/cja-b2b-edition" newtab=true tooltip="Customer Journey Analytics B2B Edition"}
* **총 인원(프로필 + 이벤트)**: 프로필 범위 인원과 이벤트 범위 인원을 결합한 유니온 수입니다.

프로필 데이터 세트의 필드를 사용하여 세 가지 범위 중 하나로 사용자 지정 지표를 만들 수도 있습니다.

## 요구 사항, 사전 요구 사항 및 고려 사항

전체 모집단 보고가 제대로 작동하려면 사전 요구 사항, 요구 사항 및 고려 사항이 고려되어야 합니다.

### 연결 요구 사항

총 모집단 보고를 지원하는 연결의 경우:

* 연결에 하나 이상의 이벤트 데이터 세트가 필요합니다. 프로필 전용 연결은 지원되지 않습니다.
* 연결에 프로필 데이터 세트를 하나 이상 추가해야 합니다. 총 모집단 보고 지표는 이벤트 데이터만 포함하는 연결에 빌드된 데이터 보기에 표시되지 않습니다.
* 공유 조회는 각 프로필 데이터 세트에 대해 구성해야 합니다. 공유 조회는 일치하는 키, 네임스페이스(ID 맵 필드의 경우) 및 조인 경로를 지정하여 각 프로필 데이터 세트가 연결의 이벤트에 조인되는 방법을 정의합니다.

#### 프로필 데이터 세트 구성

프로필 데이터 세트가 연결에 추가되면 Customer Journey Analytics이 데이터 세트 유형을 기반으로 하는 기본 공유 조회 구성을 채웁니다.

* 개인 프로필 데이터 세트의 경우: 기본적으로 ID 맵을 키 필드로 사용하여 컨테이너별 일치를 [!UICONTROL Person]&#x200B;(으)로 설정합니다. 이 기본값을 편집할 수 있습니다. 예를 들어 기본 키가 아닌 ID 맵에서 특정 네임스페이스를 선택하려면 다음과 같이 하십시오. 또는 첫 번째 네임스페이스가 채워지지 않은 경우(결합된 데이터 세트와 공통)에 대한 보조 네임스페이스를 지정합니다.
* 계정 프로필 데이터 세트 [!BADGE B2B edition]{type=Informative url="https://experienceleague.adobe.com/ko/docs/analytics-platform/using/cja-overview/cja-b2b/cja-b2b-edition" newtab=true tooltip="Customer Journey Analytics B2B Edition"}의 경우: 기본값은 컨테이너별 일치 집합입니다. [!UICONTROL 계정]&#x200B;(또는 연결에서 글로벌 계정이 사용하도록 설정된 경우 [!UICONTROL 글로벌 계정]). 계정 필드는 단일 식별자 또는 ID 맵일 수 있습니다. 계정 필드가 ID 맵인 경우 사용할 네임스페이스를 선택합니다.

단일 프로필 데이터 세트에 대해 여러 공유 조회를 구성하여 이벤트에 대한 여러 조인 경로를 지원할 수 있습니다. 동일한 ID 맵이 여러 공유 조회에서 키 필드로 사용되는 경우 네임스페이스 선택 사항이 일관되어야 합니다.

### 데이터 보기 요구 사항

데이터 보기에서 총 모집단 보고 지표가 올바르게 작동하는 경우:

* 연결에 프로필 데이터 세트가 포함되어야 합니다([연결 요구 사항](#connection-requirements) 참조). 연결에서 마지막 남은 프로필 데이터 세트를 제거하면 총 인구 보고 지표를 데이터 보기에서 사용할 수 없습니다.
* [데이터 보기 수준 세그먼트](/help/data-views/create-dataview.md#settings-segments)은(는) event-explicit가 아니어야 합니다. 데이터 보기에 직접 적용된 세그먼트가 이벤트 범위 조건(예: `hit where page = X`)으로 완전히 정의된 경우 해당 데이터 보기에서 총 모집단 보고를 수행할 수 없습니다. 총 모집단 보고 지표에 의존하기 전에 모든 데이터 보기 수준 세그먼트가 프로필 범위 보고와 호환되는지 확인하십시오.
* 지표 범위를 올바르게 설정해야 합니다. 데이터 보기 빌더에서 사용자 지정 지표 구성 요소를 만들 때 필드의 데이터 세트와 지표가 어떻게 작동하는지 기반으로 적절한 범위(이벤트, 프로필 또는 프로필 + 이벤트)를 선택하십시오. 지표를 대상 또는 반복 보고서에 사용한 후에는 이러한 종속성을 손상하지 않고 범위를 변경할 수 없습니다.

### Workspace 호환성

총 모집단 보고 지표는 대부분의 Workspace 컨텍스트에서 이벤트 기반 지표와 함께 사용할 수 있습니다. [자유 형식 테이블](/help/analysis-workspace/visualizations/freeform-analysis-visualizations.md), [라인](/help/analysis-workspace/visualizations/line.md), [막대](/help/analysis-workspace/visualizations/bar.md) 및 [가로 막대](/help/analysis-workspace/visualizations/horizontal-bar.md) 시각화, [집단 테이블](/help/analysis-workspace/visualizations/cohort-table/cohort-analysis.md) 등 적절하게 구성된 경우. 일부 시각화 유형은 기본적으로 이벤트 시퀀스에 의존하기 때문에 지원되지 않습니다.

* [폴아웃](/help/analysis-workspace/visualizations/fallout/fallout-flow.md)
* [플로우](/help/analysis-workspace/visualizations/c-flow/flow.md)
* 게시 전에 확인할 지원되지 않는 추가 시각화 유형입니다.

지원되지 않는 시각화에 총 인구 보고 지표를 추가하면 Workspace은 해당 컨텍스트에서 지표를 사용할 수 없음을 나타냅니다.

### 대상 고려 사항

총 모집단 보고 지표를 기반으로 구축된 대상은 데이터 보기에 남아 있는 해당 지표에 따라 다릅니다.

* 총 인구 보고 지표를 사용하는 반복 대상은 실패하고 데이터 보기에서 총 인구 보고 지표를 제거하면 ERROR 상태로 이동합니다.
* Customer Journey Analytics 인터페이스는 지표가 제거되지 않도록 하지만 모든 활성 반복 대상은 지표에 따라 다르며 제거를 확인하기 전에 종속성에 대한 지침을 표시합니다.

### 권한

확인: 총 모집단 보고 지표가 표시되기 전에 고객의 IMS 조직 또는 제품 프로필에 대한 모든 역할 또는 기능 액세스 요구 사항입니다. 게시하기 전에 PM에 플래그 지정할 가치가 있습니다.
